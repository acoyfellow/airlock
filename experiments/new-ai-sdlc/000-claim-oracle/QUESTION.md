# 000 — claim oracle

## Falsifiable question

Can one event-derived oracle reject a fake fleet speedup before any real model fleet is
trusted?

A useful oracle must accept one complete control run and turn red when a mutation removes
workers, disguises scripts as agents, weakens acceptance, introduces hidden human product
writes, drops commit dispositions, loses accepted behavior, breaks proof binding, stops
the clock early, omits retries, or tampers with the event chain.

## Why this comes first

A 1,000-agent visualization is easy to manufacture. Evidence that all 1,000 workers did
useful, conserved work and reached an equivalent verified preview is the scarce part.
Until this oracle is proven sensitive, worker count and PASS output are not evidence.

## Result boundary

This experiment proves only that the first mutation corpus is detected. It does not prove
real model-agent fan-out, useful scaling, deployment, or the speed thesis.
